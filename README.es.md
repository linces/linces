# 👋 Hola, soy Linces Marques

> **Desarrollador Delphi & Go** • Sistemas robustos desde 1998 • Go en producción • Delphi en el día a día

Construyo software comercial robusto desde hace casi 3 décadas. **Delphi/Object Pascal sigue siendo mi lenguaje principal para el trabajo diario**, mientras que **trabajo intensamente con Go/Golang en producción** para APIs de alto rendimiento, microservicios y modernización estratégica de sistemas legacy.

🔹 **Stack activo:** Delphi (core) • Go (producción) • Python (automatización) • PostgreSQL/Firebird/MySQL/Oracle • Docker • gRPC/REST  
🔹 **Dominios de actuación:** Licenciamiento de software • Firma digital ICP-Brasil • Pagamentos PIX • Integraciones fiscales (NFe/NFCe) • Web Scraping & Automatización

---

## 🚀 Go/Golang: Producción, Rendimiento y Modernización

- **APIs REST & gRPC en producción**: Servicios stateless con Gin, Echo y chi, enfocados en baja latencia y alta concurrencia
- **Concurrencia nativa aplicada**: Goroutines, channels y context para procesamiento asíncrono, workers y pipelines de datos
- **CLI Tools operativas**: Utilitarios en Go para automatización de deploy, validación de licencias, health checks y monitoreo
- **Integración con legacy Delphi**: Puentes vía gRPC, REST o mensajería (RabbitMQ/Kafka) para coexistencia armónica
- **Seguridad aplicada**: Middleware para JWT, OAuth2, rate limiting, logging estructurado (zerolog/logrus) y criptografía
- **Observabilidad**: Métricas con Prometheus, tracing con OpenTelemetry, logs estructurados para producción

## 💜 Delphi: Mi Herramienta Diaria (producción activa)

- **Mantenimiento & Evolución**: Refactorización segura de sistemas VCL/FireMonkey con FireDAC, Horse y ACBr
- **Nuevos Módulos**: Desarrollo de features en Delphi con arquitectura limpia y testing
- **Integraciones**: APIs REST, SOAP, gRPC y mensajería conectando Delphi con microservicios
- **Rendimiento**: Optimización de queries, caching y procesamiento asíncrono en sistemas existentes
- **Soporte Corporativo**: Sistemas críticos en producción que exigen estabilidad y conocimiento profundo de la plataforma

> 💡 *Filosofía:* "Si funciona en Delphi y entrega valor, se mantiene. Si necesita escalar o integrarse, Go entra como complemento estratégico."

## 🐍 Python: Automatización & Prototipado (soporte al stack principal)

- **Web Scraping**: BeautifulSoup, Scrapy, Selenium • Crawlers resilientes con retry y rotación de proxies
- **Automatización de Tareas**: Scripts para ETL ligero, procesamiento de archivos y glue code entre sistemas
- **IA Local & Privacidad**: Ejecución de modelos LLM localmente (llama.cpp, Ollama) con control total de los datos
- **Prototipado Rápido**: Validación de conceptos antes de reescribir en Go o integrar con Delphi

> 💡 *Estrategia:* Python para explorar → Go para producir a escala → Delphi para mantener el core estable.
